## 1. Проблема

В ходе разработки возникла необходимость совместить в одном приложении две разнородные функции: воспроизведение звука и конвертацию аудиофайлов в разные форматы.

#### Основные сложности:

- Библиотека `Qt Multimedia` (класс `QMediaPlayer`) отлично справляется с проигрыванием, но не имеет встроенных средств для глубокой конвертации форматов.

- Утилита `FFmpeg` является мощным инструментом конвертации, но работает как внешняя программа через командную строку, а не как объект C++.

- Интерфейс главного окна (`MainWindow`) не должен «знать» о деталях реализации запуска внешних процессов или специфике работы аудио-движка. Требуется универсальный способ управления звуком и конвертацией.

## 2. Решение

Для решения проблемы в проекте реализован порождающий паттерн проектирования Adapter (Адаптер).

#### Суть паттерна

Адаптер позволяет объектам с несовместимыми интерфейсами работать вместе. В данном проекте он выступает «переводчиком» между требованиями интерфейса плеера и возможностями конкретных библиотек.

#### В проекте реализованы следующие компоненты:
1. *Target (Цель)*: Абстрактный класс `AudioEngineInterface`. Он определяет единый набор команд для управления звуком (`play`, `stop`, `seek`) и функцию `convert`.

2. *Adapter (Адаптер)*: Класс `QtMultimediaAdapter`. Он реализует методы интерфейса, используя внутри себя:

- Adaptee 1 (Адаптируемый объект): Класс `QMediaPlaye` для управления звуком.

- Adaptee 2 (Внешний инструмент): Утилиту `FFmpeg` (через `QProcess`) для выполнения конвертации.

## 4. Вывод
Использование паттерна Adapter позволило:

1. Успешно интегрировать внешнюю консольную утилиту FFmpeg в объектно-ориентированную структуру C++ приложения.

2. Разделить логику управления мультимедиа и логику пользовательского интерфейса.

3. Обеспечить расширяемость системы: приложение готово к поддержке новых аудио-движков или инструментов обработки без переписывания UI-слоя.
