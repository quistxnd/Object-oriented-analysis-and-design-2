## 1. Проблема

Проблема предметной области заключается в необходимости хранения единого значения счётчика кликов, которое должно быть одинаковым на всех формах приложения.

Если каждая форма создаёт собственный экземпляр класса счётчика, возникают следующие проблемы:

- значения счётчика на формах различаются;

- данные не синхронизированы;

- изменение на одной форме не отражается на другой;

- нарушается принцип единого источника данных.

Таким образом, требуется механизм, который:

- гарантирует существование только одного экземпляра счётчика;

- предоставляет глобальный доступ к нему;

- сохраняет согласованность данных во всём приложении;

## 2. Решение

Для решения проблемы в проекте реализован порождающий паттерн проектирования Singleton (Одиночка).

### Суть паттерна

Singleton гарантирует, что:

- у класса существует только один экземпляр

- доступ к нему осуществляется через статическое свойство

  В проекте создан класс ClickCounter, который:

- содержит приватное статическое поле `_instance`;

- имеет приватный конструктор;

- предоставляет публичное статическое свойство `Instance`;

- хранит текущее значение счётчика.

#### Таким образом:
- обе формы используют один и тот же объект;

- значение счётчика синхронизировано;

- данные централизованы.
